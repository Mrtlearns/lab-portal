# Exercise 03 — Debugging with the ABAP Debugger

> [!NOTE]
> Rewritten for A4H (ABAP 2023 Docker)

**Scenario**

A provided program contains a logic error that produces incorrect output. You will use the ABAP Debugger to step through the code, inspect variable values at runtime, set breakpoints, and identify the bug — then fix it.

Prerequisites:
- **System:** A4H client 001
- **User:** `TRAIN-01` with developer authorisation
- **Exercise 02 completed** — comfortable running programs from SE38

## Task 1: Open the buggy program

1. Call transaction `SE38`.
2. Open program `Z_TRAIN_BUGGY` (pre-created in the `ZTRAINING` package).
3. Read through the source code — it is a simple loop that is supposed to sum numbers 1 to 10 and write the total.
4. Run it with **F8** and note the incorrect output.
5. Return to the editor.

## Task 2: Set a breakpoint and start the debugger

1. In the source code, locate the `LOOP` statement.
2. Click in the left margin next to the `LOOP` line — a red stop icon (breakpoint) appears.
3. Press **F8** to run the program. Execution pauses at your breakpoint and the **ABAP Debugger** opens.
4. Identify the four main debugger areas:
   - **Source code panel** (top left)
   - **Variable display** (bottom)
   - **Call stack** (top right)
   - **Toolbar** (step controls)

## Task 3: Step through the code and inspect variables

1. Press **F5** (Step Into / Single Step) to advance one line at a time.
2. After each step, observe the **Variable Display** panel:
   - Watch the loop counter variable change with each iteration.
   - Watch the accumulator variable.
3. After 3 iterations, compare the accumulator value with what you would expect mathematically.
4. Identify at which line the value goes wrong.

> [!TIP]
> To watch a specific variable, double-click its name in the source code — it is added to the **Watchpoint** list automatically.

## Task 4: Fix the bug and verify

1. Press **F12** to stop the debugger and return to the editor.
2. Based on what you observed, correct the logic error in the source code.
3. Common bugs to look for:
   - Off-by-one errors (loop starting at 0 instead of 1)
   - Wrong operator (`-` instead of `+`)
   - Variable not initialised before the loop
4. Activate the corrected program with **Ctrl+F3**.
5. Run it with **F8** — confirm the output now shows `Sum: 55`.

> [!NOTE]
> The sum of integers 1 through 10 is always 55. If your output differs, there is still a bug — step through again and check the initialisation of your accumulator variable before the loop starts.
