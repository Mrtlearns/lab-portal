# Exercise 02 — Creating a Simple ABAP Report

> [!NOTE]
> Rewritten for A4H (ABAP 2023 Docker)

**Scenario**

You will create your own executable ABAP report from scratch, add a selection screen, read data from a standard SAP table, and output a formatted list. This exercise covers the fundamental structure every ABAP developer uses daily.

Prerequisites:
- **System:** A4H client 001
- **User:** `TRAIN-01` with developer authorisation
- **Exercise 01 completed** — familiar with SE80 navigation

## Task 1: Create a new program

1. Call transaction `SE38`.
2. In the **Program** field enter `Z_TRAIN_USERLIST` and choose **Create**.
3. In the attributes dialog:
   - **Title:** `Training — User List Report`
   - **Type:** `1` (Executable Program)
   - **Status:** `T` (Test Program)
   - **Package:** `ZTRAINING`
4. Choose **Save**. A transport dialog may appear — choose **Local Object** (or assign to a transport if prompted by your instructor).
5. The ABAP editor opens with an empty `REPORT` statement.

## Task 2: Add a selection screen

After the `REPORT` line, add the following code:

```abap
SELECTION-SCREEN BEGIN OF BLOCK b1 WITH FRAME TITLE TEXT-001.
  SELECT-OPTIONS: s_uname FOR sy-uname.
SELECTION-SCREEN END OF BLOCK b1.
```

Save with **Ctrl+S**. The editor will underline `TEXT-001` — we will define this text element next.

1. Go to **Goto → Text Elements → Selection Texts**.
2. For `TEXT-001` enter: `User Selection`.
3. Save and go back to the source.

## Task 3: Read data and write output

Add the following code after the selection screen block:

```abap
DATA: lt_users TYPE TABLE OF usr02,
      ls_user  TYPE usr02.

SELECT * FROM usr02
  INTO TABLE lt_users
  WHERE bname IN s_uname.

WRITE: / 'User Name', 20 'Last Logon Date', 40 'Lock Status'.
ULINE.

LOOP AT lt_users INTO ls_user.
  WRITE: / ls_user-bname,
           20 ls_user-trdat,
           40 ls_user-uflag.
ENDLOOP.

WRITE: / sy-dbcnt, 'users found.'.
```

## Task 4: Activate and run

1. Press **Ctrl+F3** to **Activate** the program. Fix any syntax errors highlighted in red.
2. Press **F8** to **Execute**.
3. On the selection screen, leave the user name range blank (all users) and press **F8**.
4. Review the output list — confirm it shows SAP user accounts with their last logon dates.
5. Re-run with `s_uname` set to `TRAIN*` — confirm the list filters correctly.

> [!TIP]
> `sy-dbcnt` contains the number of rows returned by the last `SELECT` statement. Always display it so users know how many records were found.
